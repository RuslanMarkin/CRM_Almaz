# Публичное развёртывание в интернете

Приложение работает без логина и пароля. Все посетители видят и изменяют один
общий набор данных в MySQL. Открытые страницы автоматически обновляют данные
примерно раз в 10 секунд, а при открытии новой сессии получают актуальные данные
с сервера.

## Вариант: Railway

1. Загрузите папку `logistics-docs` в GitHub-репозиторий.
2. В Railway создайте новый проект и добавьте сервис MySQL.
3. Добавьте сервис приложения из GitHub-репозитория.
4. Если репозиторий содержит внешнюю папку `logistics-docs-source`, задайте для
   приложения Root Directory: `/logistics-docs`.
5. В Variables сервиса приложения добавьте:

   ```env
   DATABASE_URL=${{MySQL.MYSQL_URL}}
   ```

6. В Settings приложения создайте публичный домен.

Railway автоматически найдёт `Dockerfile`, соберёт приложение и проверит
`/api/health`. При каждом запуске приложение само применяет недостающие миграции.

## Другой Docker-хостинг

Нужны только Docker и постоянная MySQL 8+:

```bash
docker build -t logistics-docs .
docker run --rm -p 3000:3000 \
  -e DATABASE_URL='mysql://user:password@host:3306/logistics_docs' \
  logistics-docs
```

Хостинг должен сохранять MySQL между перезапусками. Локальная база внутри
контейнера для этого не подходит.

## Важно

Публичный режим означает, что любой человек, знающий адрес сайта, сможет
создавать, редактировать и удалять документы. Для рабочего использования
рекомендуется ограничить доступ хотя бы закрытой ссылкой, VPN или защитой на
уровне прокси, даже если отдельные аккаунты пользователям не нужны.
